## Project Overview

AI Shop Blueprint — a blog about incorporating LLMs with Etsy and Print-on-Demand businesses. Built with Astro 6, styled with Tailwind CSS v4, deployed to Cloudflare Pages as a static site.

## Commands

```bash
npm run dev          # Start dev server at localhost:4321
npm run build        # Build for production to ./dist/
npm run preview      # Preview production build
npm run deploy       # Build + deploy to Cloudflare Pages
```

## Architecture

### Tech Stack
- **Framework**: Astro 6 with static prerendering (`output: 'static'`)
- **Styling**: Tailwind CSS v4 with custom theme in `src/styles/global.css`
- **Deployment**: Cloudflare Pages (static) — no SSR adapter needed
- **Content**: Astro Content Collections with glob loader

### Routing

Simple flat routing (English only, no i18n):
- `/` — Homepage
- `/blog` — Blog listing
- `/blog/[slug]` — Individual blog post
- `/blog/tags/[tag]` — Posts filtered by tag
- `/rss.xml` — RSS feed

### Component Structure

- `src/components/layout/` — Navigation, Footer
- `src/components/seo/` — Head, JsonLd
- `src/components/ui/` — DarkModeToggle
- `src/data/` — Schema.org JSON-LD generators
- `src/layouts/` — BaseLayout.astro

### Content Collections

Blog posts managed via Astro Content Collections:

- Schema defined in `src/content.config.ts`
- Posts located in `src/content/blog/`
- Frontmatter: title, publishedAt, description, author, image, tags

### Future Post Scheduling

Posts with `publishedAt` dates in the future are automatically filtered out:
- Blog listing page (`/blog`) hides future posts
- Individual post URLs are not generated for future posts
- RSS feed excludes future posts
- Posts appear automatically once the date passes AND the site is rebuilt
- **Important**: Since this is a static site, a rebuild/deploy must happen after the publish date for the post to go live. Either run `npm run deploy` manually or set up a scheduled daily rebuild (e.g. Cloudflare Pages deploy hook on a cron)

### SEO

Each page includes:
- `<Head>` component for meta tags, Open Graph, Twitter Cards, canonical URL
- `<JsonLd>` component with Schema.org structured data
- Schema generators in `src/data/schemas.ts`
- RSS autodiscovery link in head
- Sitemap auto-generated by `@astrojs/sitemap`

### Dark Mode

Toggled via `.dark` class on `<html>`:
- Respects `prefers-color-scheme` as default
- Manual toggle persists to `localStorage`
- Works with Astro view transitions via `astro:after-swap` event
- **Critical**: Tailwind CSS v4 defaults to `@media (prefers-color-scheme: dark)` for `dark:` variants. This project uses class-based dark mode, so `global.css` must include `@custom-variant dark (&:where(.dark, .dark *));` to make `dark:` utilities respond to the `.dark` class instead of the media query. Without this, `dark:text-white` etc. will fire based on OS theme, not the toggle.

### View Transitions

Uses Astro's `<ClientRouter />` for smooth SPA-like page transitions.

### Styling Conventions

Theme defined in `src/styles/global.css`:
- Colors: Neutral grays (hex overrides in `@theme` to avoid Tailwind v4 oklch contrast issues) + indigo accent
- Fonts: DM Sans (body), Instrument Serif (display headlines) — loaded via Google Fonts `<link>` in `BaseLayout.astro`, NOT via CSS `@import` (which breaks due to Tailwind's `@import "tailwindcss"` expanding first)
- Dark mode custom styles use `.dark` class selectors (e.g. `.dark body`, `.dark .prose-blog`)

## Deployment

### Cloudflare Pages

The site is deployed to Cloudflare Pages using `wrangler`.

#### Environment Variables

These are set in `~/.zshenv`:

```bash
export CLOUDFLARE_API_TOKEN="your-api-token"
export CLOUDFLARE_ACCOUNT_ID="your-account-id"
```

#### Deploy Commands

```bash
# Build + deploy (recommended)
npm run deploy

# Or manually:
npm run build
npx wrangler pages deploy dist --project-name=cloud-blog

# First-time setup (creates the project if it doesn't exist)
npx wrangler pages project create cloud-blog --production-branch=main
```

The site will be available at: `https://cloud-blog-2jk.pages.dev`

## Creating Blog Posts

### How to create a new post

1. Create a new `.md` file in `src/content/blog/` with a URL-friendly slug as the filename (e.g. `my-new-post.md`)
2. Add the required frontmatter (see format below)
3. Write the post body in Markdown below the frontmatter
4. The post will appear on the site after the next build, provided `publishedAt` is not in the future

### Scheduling future posts

Set `publishedAt` to a future date. The post will be hidden from all pages until that date passes and the site is rebuilt. This means you can write and commit posts ahead of time.

Example: A post with `publishedAt: 2026-05-01` committed today will not appear until May 1st (after a rebuild).

### Frontmatter format

```yaml
---
title: 'Your Title Here'
publishedAt: YYYY-MM-DD
description: 'A compelling description under 160 characters'
tags: ['tag1', 'tag2']
---
```

**Required fields:**
- `title` — Post title (use single quotes; if the title contains an apostrophe, use double quotes instead)
- `publishedAt` — Date in YYYY-MM-DD format (no quotes needed)
- `description` — Short description under 160 chars for SEO/previews (use double quotes if it contains apostrophes)
- `tags` — Array of tags for categorization (lowercase, hyphenated)

**Optional fields:**
- `author` — Author name (defaults to "AI Shop Blueprint")
- `image` — URL or path to a hero image

### YAML gotcha

If your title or description contains an apostrophe (`'`), you must use double quotes:
```yaml
# Bad — YAML parse error:
description: 'Tips that aren't obvious'

# Good:
description: "Tips that aren't obvious"
```

### File naming

Use lowercase, hyphenated slugs: `my-post-title.md`. The filename becomes the URL slug (`/blog/my-post-title/`).

## Blog Post Style Guide

When creating or editing blog posts, follow this style:

**DO:**
- Write in a conversational, knowledgeable tone
- Start with a hook that connects with Etsy sellers / POD entrepreneurs
- Provide actionable, specific advice (tool names, prompt examples, workflows)
- Back up claims with data or real examples where possible
- Include practical tips readers can implement immediately
- Target 600-800 words per post
- Use tags to categorize content

**DON'T:**
- Be overly promotional or hype-driven about AI
- Ignore the legal/ethical dimensions of AI in ecommerce
- Write generic content that could apply to any business
- Use excessive jargon without explanation

## Tailwind v4 Gotchas

These are specific to this project and easy to forget:

1. **Dark mode must use class strategy**: Add `@custom-variant dark (&:where(.dark, .dark *));` in `global.css`. Without this, `dark:` utilities use `prefers-color-scheme` media query instead of the `.dark` class toggle, causing light/dark mismatch.

2. **Gray scale uses oklch by default**: Tailwind v4's default grays have a blue-ish chroma that looks washed out on warm backgrounds. This project overrides them with hex values in `@theme` (e.g. `--color-gray-900: #111827`).

3. **Font loading via `<link>`, not CSS `@import`**: `@import "tailwindcss"` expands before other rules, pushing any `@import url(...)` below it — which is invalid CSS. Fonts must be loaded via `<link>` tags in `BaseLayout.astro`.

4. **Avoid fade-in animations on content pages**: Astro view transitions (`ClientRouter`) can cause animations to re-trigger or flash on navigation. Keep animations off critical content like headings and body text.
